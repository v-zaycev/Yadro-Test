# Симулятор энергосети

Решение тестового задания — симулятор распределения энергии между генераторами и потребителями с оптимизацией стоимости и максимизацией количества включённых потребителей при дефиците.

## Допущения (относительно исходного условия)

1. Дискретность работы оборудования

   Каждый генератор может находиться только в одном из двух состояний:
   либо полностью включён (выдаёт свою номинальную мощность), либо полностью
   выключен. Частичная загрузка не допускается.

   Каждый потребитель может быть либо полностью подключён (получает весь
   запрошенный объём энергии), либо полностью отключён. Частичное питание
   не предусмотрено.

2. Целочисленность всех величин

   Все значения спроса, генерации и стоимости являются целыми числами.

   Обоснование: любые физические величины могут быть выражены в некоторой
   минимальной единице измерения (1 кВт·ч, 1 копейка, 0.1 кВт и т.д.) с
   округлением до целого числа. Выбор подходящего масштаба позволяет
   представить любые дробные значения как целые без потери точности.

3. Типы генераторов

   - flexible: переменная генерация по часам (солнечные панели, ветряки)
   - fixed: постоянная генерация (дизели, ТЭЦ)

   Оба типа имеют стоимость cost_per_kwh. Для flexible генераторов цена могла бы также быть фиксированной (не зависеть от объёма генерации), это не влияет на ход решения.  

4. Независимость часов

   Задача решается независимо для каждого часа. Не учитываются:
   - время запуска/остановки генераторов
   - накопители энергии (аккумуляторы)
   - перетоки энергии между часами

## Алгоритм решения

### Общая схема

Задача решается независимо для каждого часа (0..23). Для каждого часа
определяется суммарный спрос всех потребителей и суммарная доступная
генерация всех генераторов. В зависимости от соотношения этих величин
применяется один из двух режимов.

### Режим 1: Избыток генерации (суммарная генерация >= суммарный спрос)

Цель: минимизировать стоимость генерации при обязательном обеспечении
всех потребителей.

#### Алгоритм (динамическое программирование)

1. Формируется список генераторов для данного часа, дл:
   - мощность = hourly_output[час] (для fixed — constant_output)
   - стоимость = мощность * cost_per_kwh

2. Задача: набрать суммарную мощность >= total_demand с минимальной
   суммарной стоимостью, используя каждый генератор не более одного раза.

3. Решение через ДП:
   - costs[m] = минимальная стоимость для достижения мощности m
   - dp[0] = 0, остальные = INF
   - cost[m] = min(cost[m], cost[m-p] + cost_p)
   - ответ: min(cost[m]) для m от total_demand до max_power

4. Восстановление ответа:
   - last_gen[m] — индекс последнего добавленного генератора, для получения минимальной суммы при данной генерации
   - после получения значения m генерации, которое оптимально по стоимости для покрытия спроса, последовательность генераторов восстанавливается через спуск от m к 0 по массиву last_gen, где на каждом шаге получается последний добавленный генератор для получения данного потребления


### Почему ДП

Я рассматривал 2 варианта - перебор или динамика. В рамках тестовых примеров ДП избыточно, поскольку рассматривается малое число генераторов, но в реальных задачах уже при 40+ генераторах временная сложность при переборе становится неприемлемой, поскольку она будет O(2^n) т.е. порядка 10^12. 

В случае с динамикой временная сложность будет O(n * max_sum_power), и проблемы могут возникать только если генераторы имеют сильно разный порядок (к примеру, 100 ГВт и 1 Вт), но в реальности такое невозможно. Единственный минус динамики - сложность по памяти как O(max_sum_power), но если мы считаем, что мощности генераторов не отличаются на 6+ порядков, то это не будет существенной проблемой.

В случае реального симулятора мы можем в зависимости от условий выбирать метод решения - через дп или через перебор. Для малого числа генераторов использовать перебор, а для большого - динамику. При этом можно предварительно вычислять сложность обоих методов, и на основе данных оценок делать выбор. 

### Режим 2: Дефицит генерации (суммарная генерация < суммарный спрос)

Цель: максимизировать количество включённых потребителей. Среди вариантов
с максимальным количеством — минимизировать стоимость генерации.

### Алгоритм

1. Сортировка потребителей по возрастанию спроса
   (жадный алгоритм оптимален для максимизации количества).

2. Набираем потребителей с минимальным спросом, пока суммарный спрос не превышает максимальную генерацию.

3. Для выбранных потребителей определяем генераторы, которые покрывают их спрос используя Режим 1

### Почему жадный выбор потребителей оптимален

Для максимизации количества потребителей при фиксированной генерации
необходимо брать потребителей с самым низким спросом, поскольку любая замена одного из них на потребителя с более высоким спросом лишь увеличивает спрос при том же числе потребителей.

## Использование ИИ

В процессе разработки использовался Rubber Duck Debugging — проговаривание кода и архитектурных решений в диалоге с ИИ, что помогло рассмотреть различные подходы, а также найти более оптимальное решение. К примеру, ИИ изначально предлагал перебор, а я - динамику, но в то же время при написании динамики мой изначальный вариант был не самым оптимальным в части вычисления минимальный стоимости, а вариант ИИ - в части восстановления списка используемых генераторов. Комбинируя наши идеи удалось получить более удачный вариант.

Также ИИ использовался для следующих целей:

- Генерация чернового варианта кода (алгоритмы в основном переписывались руками, а функции вывода возвращались ИИ для правок формата и структуры)
- Генерация тестовых JSON-файлов
- Написание шаблона README

# Возможные расширения модели

## Расширения, связанные с изменением условий задачи

1. Накопители энергии (аккумуляторы)
   - Переток энергии между часами (зарядился днём от солнца, отдал ночью)
   - Ограничение на ёмкость и скорость заряда/разряда
   - Эффективность накопления (не 100%, часть теряется)

2. Приоритеты потребителей
   - Критически важные объекты (больницы, аэропорты) отключаются в последнюю очередь
   - Категории надёжности (первая, вторая, третья категория)
   - Договорная мощность (обязательство обеспечить определённый минимум)

3. Сетевые ограничения
   - Пропускная способность линий электропередач
   - Потери при передаче (зависят от расстояния)
   - Топология сети (какие генераторы с какими потребителями связаны)

4. Смешанные режимы работы генераторов
   - Дизель может работать не на полную мощность (частичная загрузка)
   - Оптимальная загрузка двигателя
   - Графики планово-предупредительных ремонтов

## Расширения, связанные с изменением самого симулятора

1. Использование других подходов для решения задачи
   - Использование готовых библиотек вместо самописного решения для целочисленного варианта
   - Симплекс-метод или его более современные аналоги при добавлении частичной загрузки

2. Многопоточность и распределённые вычисления
   - Параллельный расчёт разных часов (24 независимых задачи)
   - Распределение перебора подмножеств

3. Экономический анализ
   - Расчёт окупаемости нового оборудования
   - Сравнение сценариев (только дизели vs дизели + солнечные)
   - Анализ чувствительности к цене топлива

4. Выходные форматы
   - Экспорт в Excel (с формулами и графиками)
   - Генерация отчётов в PDF
   - API для интеграции с другими системами
