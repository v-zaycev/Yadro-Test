# Симулятор энергосети

Решение тестового задания — симулятор распределения энергии между генераторами и потребителями с оптимизацией стоимости и максимизацией количества включённых потребителей при дефиците.

## Структура проекта

```
Yadro-Test/
├── run.py                 # Точка входа
├── Dockerfile             # Docker-образ
├── src/
│   ├── main.py           # Запуск тестов и вывод результатов
│   ├── structs.py        # Dataclass-ы и загрузка JSON
│   ├── scheduler.py      # Алгоритмы оптимизации (ДП)
│   └── output.py         # Функции вывода результатов
├── test_cases/           # Тестовые JSON-файлы
│   ├── deficit_partial_coverage.json
│   ├── simple_fixed.json
│   └── surplus_full_coverage.json
└── README.md
```
## Требования

- Теоретически всё должно работать на Python 3.9 или выше
- Разработано и тестировалось на Python 3.13.9

## Запуск

### Локально (Python 3.9+)

```bash
python run.py
```
### Docker

Сборка образа:

```bash
docker build -t power-grid-simulator .
```
Запуск:
```bash
docker run --rm power-grid-simulator
```
Запуск с сохранением результатов:
```bash
mkdir -p output
docker run --rm -v "$(pwd)/output:/app/output" power-grid-simulator
```
## Тестовые случаи

| Файл | Описание |
|------|----------|
| deficit_partial_coverage.json | Дефицит генерации, часть потребителей отключается |
| simple_fixed.json | 10 дизельных генераторов — проверка выбора оптимального набора |
| surplus_full_coverage.json | Смешанный режим (избыток днём, дефицит вечером) |

## Алгоритм

### Режим избытка (генерации хватает на всех)

- Цель: минимизировать стоимость генерации
- Метод: динамическое программирование (рюкзак 0/1)
- Выбирается подмножество генераторов, покрывающее спрос с минимальной стоимостью

### Режим дефицита (генерации не хватает)

- Цель: максимизировать количество включённых потребителей
- Метод: жадный алгоритм (включаются самые малые потребители) + ДП для выбора генераторов
- Среди вариантов с максимальным количеством выбирается минимальная стоимость

## Формат входных данных (JSON)
```
{
  "name": "Название теста",
  "description": "Описание",
  "consumers": [
    {
      "name": "Завод",
      "hourly_demand": [100, 95, 90, ..., 80]
    }
  ],
  "generators": [
    {
      "name": "Солнечная",
      "type": "flexible",
      "hourly_output": [0, 0, 50, ..., 0],
      "cost_per_kwh": 10
    },
    {
      "name": "Дизель",
      "type": "fixed",
      "constant_output": 200,
      "cost_per_kwh": 45
    }
  ]
}
```
### Типы генераторов

- flexible: переменная генерация по часам (поле hourly_output)
- fixed: постоянная генерация (поле constant_output, преобразуется в список из 24 одинаковых значений)

## Формат выходных данных

Программа выводит в консоль:

- Почасовую статистику (стоимость, спрос, генерация, количество отключённых)
- Расписание работы генераторов (1 — включён, 0 — выключен)
- Расписание работы потребителей (1 — включён, 0 — отключён)

Дополнительно для каждого теста сохраняется JSON-файл с результатами (имя_теста_output.json).

## Использование ИИ

В процессе разработки использовался метод Rubber Duck Debugging — проговаривание кода и архитектурных решений в диалоге с ИИ, что помогло:

- Чётче сформулировать проблему
- Найти логические противоречия в постановке
- Проверить корректность ДП-переходов

Дополнительные сценарии использования ИИ:

- Генерация тестовых JSON-файлов
- Рефакторинг и проверка стиля кода
- Написание шаблона README

Примеры промптов:

Проверка сгенерированных частей: каждый блок кода тестировался на синтетических данных с ручным расчётом ожидаемого результата.

## Возможные расширения модели

- Аккумуляторы (сдвиг потребления во времени)
- Время запуска дизеля (нельзя включить мгновенно)
- Приоритеты потребителей (больница vs бытовая техника)
- Прогнозирование спроса и генерации
- Учёт линий передач (пропускная способность)
- Стоимость передачи энергии
- Экологические ограничения (CO2)

## Допущения и упрощения

- Солнечная энергия может иметь стоимость (не обязательно бесплатна)
- Генераторы работают дискретно (либо 0, либо полная мощность)
- Потребители получают либо весь спрос, либо 0 (нельзя частично)
- Нет аккумуляторов (энергия не накапливается)
- Нет времени запуска/остановки генераторов
- Все потребители равны при отключении (нет приоритетов)
- Стоимость линейна (нет скидок за объём)
